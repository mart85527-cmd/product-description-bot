# Text Genius — Руководство для Claude Code

## Общая информация о проекте

**Text Genius** — Telegram-бот на Python, который генерирует продающие описания товаров через API YandexGPT. Целевая аудитория: владельцы бизнесов на маркетплейсах (Wildberries, Ozon, Яндекс.Маркет, Авито) и интернет-магазинах.

**Важно:** перед любыми изменениями читай `config.py` и `database.py` — там сосредоточена вся конфигурация и схема данных. Не вносите изменения, которые могут сломать существующую базу данных SQLite у пользователей.

## Запуск и тестирование

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Настройка окружения

1. Скопируйте `.env.example` в `.env`
2. Заполните реальными значениями (токены, ключи, ID админа)
3. **Никогда не коммитьте `.env`** — он уже в `.gitignore`

### Запуск бота

```bash
python bot.py
```

Бот работает в режиме **polling** (опроса серверов Telegram). При первом запуске автоматически создаётся файл `bot.db` (SQLite).

### Проверка работоспособности

1. Напишите боту `/start` — должно появиться главное меню
2. Нажмите «Создать описание» — пройдите все шаги до результата
3. Проверьте команды `/help`, `/cancel`
4. Проверьте админ-команду `/add_balance <user_id> 1` (требует `ADMIN_ID` из `.env`)

## Как вносить изменения

### Добавление нового шага в диалог

Диалог реализован через **FSM** (Finite State Machine — конечный автомат) из библиотеки `python-telegram-bot`. Каждый шаг — это отдельное состояние.

**Порядок действий:**
1. В `bot.py` добавьте новое состояние в Enum `States` (или в словарь состояний)
2. Добавьте обработчик `MessageHandler` или `CallbackQueryHandler`, который переходит из предыдущего состояния в новое
3. Добавьте обработчик, который обрабатывает новое состояние и переводит в следующее
4. Обновите функцию `start` (команда `/start`), чтобы сброс возвращал пользователя в начало
5. Обновите `README.md`: раздел «Сценарий диалога (FSM)»

### Добавление новой команды

1. В `bot.py` создайте функцию-обработчик с декоратором `@app.command_handler("/команда")`
2. Если команда админская — проверяйте `user_id == int(config.ADMIN_ID)`
3. Обновите список команд в `README.md`

### Изменение промпта для YandexGPT

1. Откройте `yandex_gpt.py`
2. Найдите функцию, формирующую `system_prompt` или `user_prompt`
3. Измените текст. **Проверьте:** не нарушаете ли существующие запреты (шаблонные фразы)
4. Протестируйте генерацию на 2-3 разных товарах
5. Обновите `README.md`: раздел «Промпт-инжиниринг»

### Добавление поля в базу данных

1. Откройте `database.py`
2. Найдите функцию `init_db()` — там `CREATE TABLE` при первом запуске
3. Добавьте новое поле в `CREATE TABLE` для новых пользователей
4. Для существующих баз данных добавьте `ALTER TABLE` в функцию миграции или в `init_db()` с проверкой `PRAGMA table_info(table_name)`
5. Обновите `README.md`: таблицы в разделе «Архитектура базы данных»

**Важно:** SQLite не поддерживает удаление столбцов. Если нужно убрать поле — создайте новую таблицу, скопируйте данные, удалите старую, переименуйте новую.

## YandexGPT API — что нужно знать

- **Endpoint:** `https://llm.api.cloud.yandex.net/foundationModels/v1/completion`
- **Модель по умолчанию:** уточняйте в `yandex_gpt.py` (обычно `yandexgpt-lite` или `yandexgpt`)
- **Аутентификация:** через `Api-Key` в заголовке + `folderId` в теле запроса
- **Ограничения:** следите за лимитами токенов в ответе (`maxTokens`)
- **Промпт:** состоит из `system` (инструкции) и `user` (данные о товаре). Баланс между подробностью и краткостью — ключ к качеству

Если генерация возвращает ошибку:
1. Проверьте `YANDEXGPT_API_KEY` и `YANDEX_FOLDER_ID` в `.env`
2. Проверьте баланс в Yandex Cloud (счёт должен быть пополнен)
3. Проверьте логи — YandexGPT возвращает подробные ошибки (401, 403, 429, 500)

## Проверка ошибок и отладка

### Частые проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| `ValueError: BOT_TOKEN не найден` | Нет файла `.env` или переменная не заполнена | Создать `.env` по шаблону `.env.example` |
| `ModuleNotFoundError` | Не установлены зависимости | `pip install -r requirements.txt` |
| Бот не отвечает | Бот не запущен или токен неверный | Проверить токен в @BotFather, перезапустить |
| `503 Service Unavailable` (YandexGPT) | Прокси PythonAnywhere или проблемы Yandex Cloud | Подождать 5–15 минут, перезапустить |
| Ошибка SQLite | Повреждён `bot.db` | Переименовать/удалить `bot.db` (данные потеряются) |
| `Unauthorized` (403) | Неверный `YANDEXGPT_API_KEY` | Проверить ключ в Yandex Cloud Console |

### Как смотреть логи

При запуске `python bot.py` ошибки выводятся в консоль (stderr). Для продакшена рекомендуется добавить логирование в файл.

**Добавление логирования в файл:**
```python
import logging
logging.basicConfig(
    filename='bot.log',
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

## Правила безопасности (жёсткие ограничения)

1. **Никогда** не коммитьте `.env` — там токены и API-ключи
2. **Никогда** не коммитьте `bot.db` — там данные пользователей
3. **Никогда** не показывайте `BOT_TOKEN` или `YANDEXGPT_API_KEY` в логах или сообщениях пользователю
4. **Никогда** не храните пароли или ключи в открытом виде в коде — только через `.env`/`config.py`
5. При замене API-ключа: сразу отзовите старый в Yandex Cloud / @BotFather
6. Перед публикацией скриншотов или логов: замажьте/удали токены, ID пользователей, номера телефонов

## Стиль кода

- Функции и переменные — `snake_case` (например: `add_balance`, `price_per_description`)
- Константы в `config.py` — `UPPER_CASE`
- Комментарии — только если объясняют **почему**, а не **что** (код должен говорить сам за себя)
- Ошибки — понятные сообщения на русском языке для пользователя, технические детали — в логах
- `async/await` — весь бот асинхронный, не используйте блокирующие операции (например, `requests` вместо `httpx`)
