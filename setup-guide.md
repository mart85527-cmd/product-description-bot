# Инструкция по развёртыванию бота «Генератор описаний товаров»

Этот документ поможет вам создать точно такой же Telegram-бот, как у Марины. Следуйте шагам по порядку — и через 30–40 минут у вас будет работающий бот.

---

## Что вам понадобится до начала

| Что нужно | Где получить | Для чего |
|---|---|---|
| Telegram-аккаунт | У вас уже есть | Управление ботом, общение с пользователями |
| Токен бота | @BotFather в Telegram | Чтобы бот мог отправлять сообщения |
| Yandex Cloud аккаунт | [cloud.yandex.ru](https://cloud.yandex.ru) | Доступ к YandexGPT API |
| API-ключ YandexGPT | Yandex Cloud → API-ключи | Генерация текстов описаний |
| Folder ID | Yandex Cloud → Облако | Привязка ключа к вашему каталогу |
| Ваш Telegram ID | @userinfobot в Telegram | Чтобы бот знал, кто администратор |
| Реквизиты для оплаты | Ваш банк | Пользователи будут переводить на них деньги |
| Сервер (компьютер или облако) | PythonAnywhere или ваш ПК | Где бот будет работать 24/7 |

## Шаг 1. Создание бота в Telegram

1. Откройте Telegram и найдите контакт **@BotFather**.
2. Отправьте команду `/newbot`.
3. Придумайте название бота (например, «Генератор описаний») и username (должен заканчиваться на `bot`, например `my_text_genius_bot`).
4. BotFather пришлёт вам **токен** — строку вида `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`. **Сохраните его в надёжном месте** — это главный пароль от бота.
5. В той же переписке с BotFather отправьте `/setcommands` и выберите своего бота, чтобы добавить команды:
   ```
   start - Главное меню
   help - Справка по боту
   cancel - Отменить текущий диалог
   ```

---

## Шаг 2. Получение YandexGPT API

1. Зарегистрируйтесь на [Yandex Cloud](https://cloud.yandex.ru).
2. Перейдите в **Биллинг** и привяжите карту (не волнуйтесь — есть бесплатный стартовый грант).
3. Создайте **сервисный аккаунт**:
   - Меню → Сервисные аккаунты → Создать.
   - Название: `text-genius-bot`.
   - Роль: `ai.languageModels.user`.
4. Создайте **API-ключ** для этого сервисного аккаунта:
   - Меню → API-ключи → Создать.
   - Сохраните ключ — он показывается только один раз.
5. Скопируйте **Folder ID** (идентификатор каталога) из адресной строки или из меню «Облако».

---

## Шаг 3. Узнаём свой Telegram ID

1. Найдите в Telegram бота **@userinfobot**.
2. Отправьте ему любое сообщение.
3. Он ответит вашим ID — число, например `796743353`. Это ваш `ADMIN_ID`.

---

## Шаг 4. Скачивание и подготовка файлов

Создайте папку на компьютере, например `C:\bots\text-genius` (Windows) или `~/bots/text-genius` (Mac/Linux).

Скопируйте в неё эти файлы из проекта Марины:

- `bot.py` — главный файл бота
- `yandex_gpt.py` — работа с YandexGPT
- `database.py` — база данных
- `config.py` — чтение настроек
- `requirements.txt` — список библиотек
- `.env` — пример файла настроек (см. шаг 5)

---

## Шаг 5. Настройка файла `.env`

Создайте в папке проекта файл `.env` (именно так, с точкой в начале).

Заполните его, подставив свои данные:

```env
# Токен бота от @BotFather
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz

# YandexGPT (получили на шаге 2)
YANDEXGPT_API_KEY=AQVN... (ваш ключ)
YANDEX_FOLDER_ID=b1g... (ваш Folder ID)

# Ваш Telegram ID (узнали на шаге 3)
ADMIN_ID=796743353

# Реквизиты для оплаты — что увидит пользователь
PAYMENT_DETAILS=СБП на номер +7 (920) 803-57-56 (Марина, Т-банк)

# Ценообразование (можно менять)
PRICE_PER_DESCRIPTION=50
DISCOUNT_PACKAGE_SIZE=50
DISCOUNT_PERCENT=20
```

**Важно:** файл `.env` никогда не отправляйте в интернет и не показывайте никому. В нём ваши пароли.

## Шаг 6. Установка Python и библиотек

### Windows

1. Установите Python 3.11 или новее с [python.org](https://www.python.org).
2. Во время установки **обязательно** поставьте галочку «Add Python to PATH».
3. Откройте командную строку (`Win + R`, введите `cmd`).
4. Перейдите в папку проекта:
   ```cmd
   cd C:\bots\text-genius
   ```
5. Установите библиотеки:
   ```cmd
   pip install -r requirements.txt
   ```

### Mac / Linux

```bash
# Перейти в папку
cd ~/bots/text-genius

# Создать виртуальное окружение (рекомендуется)
python3 -m venv venv
source venv/bin/activate

# Установить библиотеки
pip install -r requirements.txt
```

## Шаг 7. Первый запуск (локально)

В командной строке, находясь в папке проекта:

```bash
python bot.py
```

Если всё правильно, вы увидите:

```
Бот запущен. Нажмите Ctrl+C для остановки.
```

**Проверьте бота в Telegram:**
1. Найдите своего бота по username.
2. Отправьте `/start`.
3. Должно появиться меню с балансом и кнопками.
4. Нажмите «Создать описание», пройдите шаги и проверьте, что текст генерируется.

Если появляется ошибка — она будет напечатана в консоли. Скопируйте текст ошибки и разберитесь (самая частая — опечатка в токене или API-ключе).

## Шаг 8. Деплой на сервер (чтобы бот работал 24/7)

Когда бот работает локально, он выключается, как только вы закрываете компьютер. Для постоянной работы нужен сервер.

### Вариант А: PythonAnywhere (бесплатно, для старта)

**Плюс:** Не нужен свой компьютер. Бесплатно.
**Минус:** Прокси иногда отваливается (ошибка 503). Нужно перезапускать.

1. Зарегистрируйтесь на [pythonanywhere.com](https://www.pythonanywhere.com).
2. Перейдите в **Files** → создайте папку `text-genius`.
3. Загрузите все файлы проекта (через кнопку Upload a file или в zip-архиве).
4. Откройте **Consoles** → **Bash**.
5. Перейдите в папку:
   ```bash
   cd text-genius
   ```
6. Установите библиотеки:
   ```bash
   pip install --user -r requirements.txt
   ```
7. Запустите:
   ```bash
   python bot.py
   ```
8. Бот работает! Но консоль закроется, если вы уйдёте с сайта.

Чтобы бот не выключался, оставьте консоль открытой или используйте **Always-on task** (платная функция).

### Вариант Б: Свой компьютер (Windows)

Если хотите запускать бот на своём рабочем компьютере:

1. Создайте `.bat` файл в папке проекта:
   ```bat
   @echo off
   cd C:\bots\text-genius
   python bot.py
   pause
   ```
2. Дважды кликните по файлу — бот запустится в чёрном окне.
3. Минус: бот работает только пока включён компьютер и открыто окно.

### Вариант В: Timeweb Cloud (платный, надёжный)

**Плюс:** Стабильно 24/7, нет ограничений прокси, дёшево.
**Минус:** Нужно платить ~200 ₽/мес и немного настроить.

Подробная инструкция в файле `timeweb-migration-guide.md` (в папке проекта).

## Шаг 9. Настройка администратора

Когда бот запущен и вы написали ему `/start`:

1. Ваш баланс = 0.
2. Чтобы начать тестировать, пополните себе баланс через команду в Telegram (отправьте себе):
   ```
   /add_balance 1
   ```
   (вместо `1` можно любое число — это количество описаний, которые вы начисляете себе).
3. Команда работает только для `ADMIN_ID` — другие пользователи не смогут ей воспользоваться.

## Шаг 10. Проверка перед запуском для клиентов

Пройдите этот чек-лист:

- [ ] Бот отвечает на `/start` и показывает меню.
- [ ] Нажатие «Создать описание» начинает диалог.
- [ ] Все 5 площадок выбираются корректно.
- [ ] Тон выбирается (стандартный и 4 варианта).
- [ ] После последнего шага генерируется текст и приходит в Telegram.
- [ ] Текст качественный, без шаблонных фраз.
- [ ] Баланс списывается (-1 за генерацию).
- [ ] При нулевом балансе бот предлагает пополнить.
- [ ] Кнопка «Пополнить баланс» показывает реквизиты и прайс.
- [ ] Кнопка «Я оплатил» отправляет уведомление вам (администратору).
- [ ] Команда `/add_balance` работает и начисляет баланс.
- [ ] Кнопка «История» показывает последние генерации.
- [ ] Кнопка «Избранное» работает (сохранение и просмотр).
- [ ] Кнопка «Сгенерировать другой вариант» создаёт новый текст.
- [ ] Команда `/help` показывает справку.
- [ ] Кнопка «Отмена» работает на каждом шаге диалога.
- [ ] Отправка `/start` во время диалога сбрасывает его.

## Как обновлять бота (если Марина добавила новую фичу)

Если проект обновился, вам нужно заменить файлы на сервере:

1. Скачайте новые файлы из репозитория.
2. На PythonAnywhere: загрузите через Files → замените старые.
3. Перезапустите бота в консоли:
   ```bash
   pkill -f bot.py
   python bot.py
   ```
4. Важно: если меняется структура базы данных (`database.py`), бот обновит таблицы автоматически при следующем запуске. Но если структура меняется сильно — может понадобиться удалить старый `bot.db` (вы потеряете историю).

## Если что-то пошло не так

### Ошибка «Invalid token» или бот не отвечает
- Проверьте `BOT_TOKEN` в `.env` — нет ли лишних пробелов или скопирован ли он полностью.

### Ошибка «YandexGPT ошибка 401»
- API-ключ неверный или у сервисного аккаунта нет прав `ai.languageModels.user`. Пересоздайте ключ в Yandex Cloud.

### Ошибка «ProxyError: 503» (PythonAnywhere)
- Прокси бесплатного тарифа временно недоступен. Подождите 10–15 минут и перезапустите `python bot.py`.

### Бот запустился, но не отвечает на сообщения
- Убедитесь, что нет другого запущенного процесса этого же бота (например, на другом сервере или в другой консоли). Telegram разрешает только одно соединение на токен.

### Баланс не списывается или не начисляется
- Проверьте, что файл `bot.db` создаётся в папке проекта. Если нет — проверьте права на запись в папку.
