# 📝 Text Genius — Генератор описаний товаров

> **MVP-версия Telegram-бота** для автоматической генерации продающих текстов товаров через YandexGPT API. Разработан для владельцев бизнесов на маркетплейсах (Wildberries, Ozon, Яндекс.Маркет, Авито) и собственных интернет-магазинов.

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| **Пошаговый диалог** | Бот последовательно запрашивает: площадку, тон, категорию, характеристики, аудиторию, преимущества, ключевые слова |
| **Генерация текста** | Через YandexGPT API создаются живые, конкретные описания без шаблонных фраз |
| **5 площадок** | Wildberries, Ozon, Яндекс.Маркет, Авито, Свой сайт (у каждой свой стиль текста) |
| **5 тонов** | Стандартный, Формальный, Дружелюбный, Агрессивный, Юмористический |
| **Баланс и списание** | 1 генерация = минус 1 описание с баланса. Настраиваемая цена и скидки через `.env` |
| **Ручное пополнение** | Кнопка «Пополнить баланс» → реквизиты СБП → «Я оплатил» → уведомление администратору |
| **История и избранное** | Последние 10 генераций в «Истории», лучшие можно сохранить в «Избранное» |
| **Перегенерация** | Кнопка «Сгенерировать другой вариант» без повторного ввода данных |
| **Админ-команды** | `/add_balance` для ручного начисления баланса пользователям |
| **Отмена и сброс** | Кнопка «Отмена» на каждом шаге, `/start` сбрасывает диалог в любой момент |

## 🛠 Стек технологий

- **Python 3.11+** — язык программирования
- **python-telegram-bot 21.10** — библиотека для работы с Telegram Bot API
- **SQLite** — встроенная локальная база данных (не требует отдельного сервера)
- **httpx 0.27.0** — асинхронные HTTP-запросы к YandexGPT API
- **python-dotenv 1.0.1** — чтение переменных окружения из файла `.env`
- **YandexGPT API** (`llm.api.cloud.yandex.net`) — генерация текста на основе промптов

## 🚀 Установка и запуск

### 1. Клонирование и установка зависимостей

```bash
# Клонировать репозиторий
git clone https://github.com/MarinaLogistics/product-description-bot.git
cd product-description-bot

# Установить зависимости
pip install -r requirements.txt
```

**Что такое `pip`?** Это менеджер пакетов Python — программа, которая скачивает и устанавливает библиотеки одной командой.

### 2. Настройка переменных окружения

Скопируйте шаблон и переименуйте:

```bash
# Windows (PowerShell)
copy .env.example .env

# Windows (Git Bash / Linux / macOS)
cp .env.example .env
```

Откройте файл `.env` в любом текстовом редакторе и заполните реальными значениями:

### 3. Запуск

```bash
python bot.py
```

Бот запустится в режиме **polling** (опроса) и будет непрерывно слушать сообщения от Telegram.

## 🔐 Настройка окружения (.env)

| Переменная | Описание | Где получить |
|------------|----------|--------------|
| `BOT_TOKEN` | Токен доступа к вашему боту | [@BotFather](https://t.me/BotFather) в Telegram — создайте бот и скопируйте токен |
| `YANDEXGPT_API_KEY` | API-ключ для доступа к YandexGPT | [Yandex Cloud](https://yandex.cloud/ru/docs/iam/operations/api-key/create) — создайте API-ключ для сервисного аккаунта |
| `YANDEX_FOLDER_ID` | ID каталога в Yandex Cloud | [Yandex Cloud Console](https://console.yandex.cloud/) — откройте каталог, ID скопируйте из адресной строки |
| `ADMIN_ID` | Telegram user ID администратора | Напишите [@userinfobot](https://t.me/userinfobot) в Telegram — бот покажет ваш ID |
| `PAYMENT_DETAILS` | Текст с реквизитами для оплаты | Укажите номер телефона/банк для СБП |
| `PRICE_PER_DESCRIPTION` | Цена одного описания в рублях | По умолчанию: `50` |
| `DISCOUNT_PACKAGE_SIZE` | Размер пакета для скидки | По умолчанию: `50` |
| `DISCOUNT_PERCENT` | Процент скидки на пакет | По умолчанию: `20` |

**Важно:** файл `.env` никогда не попадает в Git (он указан в `.gitignore`) — так ваши токены и ключи остаются в безопасности.

## 🗄 Архитектура базы данных

Используется **SQLite** — легковесная встроенная база данных, которая хранится в одном файле и не требует установки сервера.

### Таблица `users` — пользователи бота
| Поле | Тип | Описание |
|------|-----|----------|
| `user_id` | INTEGER PRIMARY KEY | Telegram ID пользователя |
| `username` | TEXT | Telegram username (без @) |
| `first_name` | TEXT | Имя пользователя |
| `balance` | INTEGER DEFAULT 0 | Баланс в количестве доступных описаний |
| `created_at` | TEXT | Дата регистрации (формат ISO 8601) |

### Таблица `generations` — история генераций
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Уникальный номер генерации |
| `user_id` | INTEGER | Ссылка на пользователя (кому принадлежит) |
| `platform` | TEXT | Площадка (Wildberries, Ozon, Яндекс.Маркет, Авито, Сайт) |
| `category` | TEXT | Категория товара |
| `tone` | TEXT | Выбранный тон текста |
| `result_text` | TEXT | Сгенерированное описание (полный текст) |
| `created_at` | TEXT | Дата и время генерации |

### Таблица `favorites` — избранное
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Уникальный номер записи |
| `user_id` | INTEGER | Ссылка на пользователя |
| `generation_id` | INTEGER | Ссылка на генерацию (из таблицы generations) |
| `created_at` | TEXT | Дата сохранения в избранное |

## 🔄 Сценарий диалога (FSM)

**FSM** (Finite State Machine, конечный автомат) — это способ организации диалога, когда бот помнит, на каком шаге находится пользователь, и ждёт конкретный ответ.

```
/start → Главное меню
    ↓ «Создать описание"
[Шаг 1] Выбор площадки (WB / Ozon / Яндекс.Маркет / Авито / Сайт)
    ↓
[Шаг 2] Выбор тона (Стандартный / Формальный / Дружелюбный / Агрессивный / Юмористический)
    ↓
[Шаг 3] Ввод категории товара
    ↓
[Шаг 4] Ввод характеристик
    ↓
[Шаг 5] Ввод целевой аудитории
    ↓
[Шаг 6] Ввод преимуществ
    ↓
[Шаг 7] Ввод ключевых слов (или «нет»)
    ↓
Генерация текста → Результат + кнопки:
    • 💾 Сохранить в избранное
    • 🔄 Сгенерировать другой вариант
    • 🏠 В главное меню
```

**Доступные команды:**
- `/start` — главное меню, сброс любого диалога
- `/help` — справка по использованию
- `/cancel` — отмена текущего диалога
- `/add_balance <user_id> <amount>` — начислить баланс пользователю (только для админа)

## 🌍 Деплой (размещение на сервере)

### Вариант 1: PythonAnywhere (бесплатный тариф)

Подходит для тестирования и небольшой нагрузки.

**Ограничения:**
- Исходящие запросы идут через общий прокси (иногда возникает ошибка `503 Service Unavailable` — нужно подождать 5–15 минут и перезапустить бота).
- Бот должен быть запущен вручную (или через «Always-on task» на платном тарифе).

**Плюсы:**
- Полностью бесплатно
- Не требует настройки Linux-сервера

### Вариант 2: Timeweb Cloud VPS (рекомендуется для продакшена)

- Стоимость: от ~200 ₽/месяц
- Стабильное соединение, нет ограничений прокси
- Полный контроль над сервером
- Подробная инструкция по переезду находится в файле `timeweb-migration-guide.md`

## 💰 Ценообразование

Вся настройка цен происходит через файл `.env` без изменения кода:

- **1 описание** = `PRICE_PER_DESCRIPTION` ₽ (по умолчанию 50 ₽)
- **Пакет со скидкой** = `DISCOUNT_PACKAGE_SIZE` описаний со скидкой `DISCOUNT_PERCENT`%
  - Пример: 50 описаний за 2000 ₽ вместо 2500 ₽ (экономия 500 ₽, скидка 20%)

Баланс хранится в количестве описаний (не в рублях), чтобы упростить логику.

## 🧠 Промпт-инжиниринг

Бот использует строгий **системный промпт** (инструкция для нейросети), который запрещает шаблонные и пустые фразы:

**❌ Запрещено:**
- «идеальный выбор», «отличный выбор», «идеально подходит»
- «порадуйте», «подарите», «радуйтесь»
- «не только..., но и...»
- Пустые комплименты без конкретных фактов

**✅ Требуется:**
- Конкретные факты (материал, вес, размер, цвет)
- Ощущения (какой на ощупь, как выглядит, как пахнет)
- Ситуации использования (где и когда пригодится)
- Прямой призыв к действию (CTA): «Закажите», «Купите», «Добавьте в корзину»
